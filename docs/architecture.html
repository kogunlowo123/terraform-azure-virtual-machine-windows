terraform-azure-virtual-machine-windows

Executive Summary

This Terraform module deploys Azure Windows Virtual Machines with comprehensive enterprise features. It includes automatic NIC and public IP creation, managed OS and data disks with encryption options, availability zone and availability set placement, proximity placement groups, Trusted Launch (Secure Boot + vTPM), built-in extensions (Microsoft Antimalware, Azure Monitor Agent, Azure AD Login), custom extensions, WinRM listener configuration, boot diagnostics, managed identity, Azure Hybrid Benefit licensing, configurable patching modes, and Azure Backup integration. The module supports both marketplace images and custom image IDs.

Component Breakdown

Windows Virtual Machine

Core compute with configurable size (Standard_D2s_v3 default), admin credentials, computer name, source image (marketplace or custom ID), license type for Hybrid Benefit, automatic updates, patch mode (Manual/AutomaticByOS/AutomaticByPlatform), hotpatching, timezone, WinRM listeners, unattend content, and Trusted Launch (Secure Boot + vTPM).

Network Interface and Public IP

Auto-creates NIC when no network_interface_ids provided. Optional static public IP (Standard SKU) with zone awareness. Supports Dynamic/Static private IP allocation.

Data Disks

Map-based managed disks with configurable storage type (Premium_LRS), size, caching, LUN, encryption set, tier, and per-disk zone override.

Extensions

Antimalware: Microsoft IaaS Antimalware with real-time protection, scheduled scans (Quick/Full), scan day/time, and file/path/process exclusions. Azure Monitor Agent: Windows monitoring agent with auto-upgrade. Azure AD Login: AAD-based authentication for RDP. Custom: Any additional VM extension via map configuration.

High Availability

Supports availability zones (1, 2, 3), availability sets (cannot combine with zones), and proximity placement groups for latency optimization.

Security Features

Encryption at host, Trusted Launch (Secure Boot + vTPM), Disk Encryption Sets, managed identity, and Azure AD login for passwordless RDP.

Data Flow

  1. VM provisions from marketplace image or custom image ID
  2. NIC is auto-created or externally provided, attached to subnet
  3. Optional public IP enables inbound connectivity
  4. Admin authenticates via password or Azure AD (with AAD Login extension)
  5. Antimalware extension provides real-time protection and scheduled scans
  6. Azure Monitor Agent collects performance and event logs
  7. Data disks are attached at specified LUNs
  8. Boot diagnostics capture serial console for troubleshooting
  9. Azure Backup takes scheduled snapshots to Recovery Services Vault
  10. Managed identity enables secure resource access without credentials

Industry Adaptation

Enterprise Applications

Windows Server 2022 with Hybrid Benefit licensing, domain join via custom extension, availability zones, antimalware, full monitoring, daily backup.

SQL Server Workloads

Premium SSD data disks with ReadOnly caching, large VM sizes, disk encryption, proximity placement groups for low latency, backup with SQL-aware policy.

Remote Desktop Services

Azure AD Login for passwordless RDP, WinRM listeners for remote management, multiple zones for HA, antimalware enabled.

Compliance Environments

Trusted Launch enabled, encryption at host, CMK disk encryption, AAD-only auth, full audit via AMA, backup with long retention, disabled public IP.

Production Readiness Checklist

  • Enable Trusted Launch (Secure Boot + vTPM)
  • Enable encryption at host for data protection
  • Configure Disk Encryption Sets with CMK
  • Enable Microsoft Antimalware extension
  • Enable Azure Monitor Agent for observability
  • Place VM in availability zone or availability set
  • Configure Azure Backup with appropriate retention
  • Use managed identity instead of stored credentials
  • Set appropriate patching mode (AutomaticByPlatform recommended)
  • Consider Azure AD Login for passwordless authentication
  • Disable public IP unless required
  • Apply resource tags for governance and cost tracking

Deployment

# Initialize Terraform
terraform init

# Preview changes
terraform plan -out=tfplan

# Apply configuration
terraform apply tfplan

Prerequisites

  • Terraform >= 1.3.0
  • AzureRM provider >= 3.80.0
  • Existing Resource Group and subnet (or external NIC)
  • For backup: Recovery Services Vault and backup policy
  • For Trusted Launch: Gen2 image support
